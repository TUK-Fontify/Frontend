Fontify 프론트엔드 API 연동 필요 명세
작성일: 2026-05-06

현재 백엔드 기준 주소
- 기본 URL: http://서버주소/api/v1
- 개발 인증 헤더: x-user-id: dev-user-001
- 프론트 환경변수:
  - VITE_API_BASE_URL=http://localhost:8000/api/v1
  - VITE_DEV_USER_ID=dev-user-001

1. 현재 백엔드로 바로 연동 가능한 API

1-1. 회원
- POST /auth/signup
- POST /auth/login
- POST /auth/dev-login
- GET /users/me
- PATCH /users/me

1-2. 마이페이지
- GET /users/me/ratings
- GET /users/me/generations
- GET /users/me/downloads

1-3. 폰트
- GET /fonts?page=1&limit=100
- GET /fonts/{font_id}
- POST /fonts/{font_id}/download
- POST /fonts/{font_id}/tag
- GET /generated_fonts?page=1&limit=20
- GET /generated_fonts/{generated_font_id}
- POST /generated_fonts/{generated_font_id}/rate
- GET /generated_fonts/{generated_font_id}/rate

1-4. 생성 작업
- POST /generations/google
  요청: { "font_file_id": 1 }
- POST /generations/handwriting
  요청: { "handwriting_id": 1 }
- GET /generations/{job_id}

1-5. 손글씨
- POST /handwriting/upload
  multipart form-data: image

2. 프론트에서 연동 준비 완료한 위치

- API 클라이언트: src/api/client.ts

4. 백엔드에서 수정이 필요한 것으로 보이는 항목

4-1. CORS 설정
Vite 개발 서버가 http://localhost:5173 이므로 백엔드에서 CORS 허용이 필요합니다.
권장 허용 origin:
- http://localhost:5173
- 배포 프론트 도메인

4-2. /fonts/{font_id} path parameter 버그
현재 코드상 라우트는 /fonts/{font_id}인데 함수 인자가 file_id로 되어 있습니다.
FastAPI path param 바인딩을 위해 아래 둘 중 하나로 맞춰야 합니다.
- def get_font(font_id: int, ...)
- 또는 라우트를 /fonts/{file_id}로 변경

4-3. 다운로드 API 이름 정리
현재 POST /fonts/{font_id}/download 는 실제로 GeneratedFont를 조회합니다.
혼동 방지를 위해 아래 형태를 권장합니다.
- POST /generated_fonts/{generated_font_id}/download

4-4. 정적 파일 서빙
file_url이 /uploads/... 또는 /폰트경로 형태로 내려오므로 FastAPI에서 StaticFiles mount가 필요합니다.
예:
- /uploads
- /fonts
- /static

4-5. 완료된 generation job과 generated_font 연결
프론트 작업 완료 화면에서 TTF 다운로드 버튼을 보여주려면 generation 응답 또는 work 응답에 downloadUrl이 필요합니다.

5. 현재 프론트 임시 매핑 규칙

- /users/me/generations의 status/progress를 프론트 phase로 임시 변환합니다.
- progress 0~14: analyzing
- progress 15~34: preview_ready
- progress 35~84: retraining
- progress 85~99: finalizing
- progress 100 또는 COMPLETED/DONE: completed
- PENDING/QUEUE: queued
- FAIL/ERROR: failed

백엔드에서 phase를 직접 내려주면 이 임시 규칙은 제거할 수 있습니다.
